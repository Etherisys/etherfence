# Human Posture Layout Contract

Applies only to `etherfence scan` human output.

- Existing section headings and canonical labels remain: `Security posture`, `Posture`, `Scope`, `Assessment`, `Priority findings`, `Why this matters`, and `Next steps`.
- A wrapped continuation aligns beneath its field/bullet content and never starts flush-left as a new risk/action.
- Priority risk and corresponding action each retain the same finding ID in their leading line.
- Colors are cosmetic. With `NO_COLOR`, redirected stdout, or unsupported color, text has no ANSI escape sequences.
- Width changes presentation only; it never affects report content, priority order, scores, grades, findings, or process exit status.
